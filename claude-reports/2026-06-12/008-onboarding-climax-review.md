## Decisions

- **First spark reuses the real `QuoteCard`** (read-only, `onShare: null`), wrapped
  in a fixed-height `ClipRRect` inside the step's scrollable body so its internal
  `Spacer`s lay out and it can't overflow. The quote is chosen deterministically
  (lowest id) from `watchQuotesByCategory(category)`. The category may be premium
  — that's the intended "taste"; nothing unlocks. If the category has no quotes,
  a gentle placeholder shows and the user continues.
- **Review prompt seam.** A `ReviewPrompter` interface wraps `in_app_review`
  (guarded by `isAvailable()`, try/caught). Tests override
  `reviewPrompterProvider` with a fake — the real plugin is never called in tests.
- **Fires once, at the peak.** The streak step takes an `active` flag; when it
  flips true (the page becomes visible), it plays confetti and starts a 1.2 s
  timer. The timer sets `reviewPromptShown = true` **before** calling
  `requestReview()`, so a crash mid-request can't double-fire, and the flag is
  re-checked so it never fires on a later run. The streak step is always
  second-to-last, so its index is stable despite the conditional journey step —
  that's how it knows when it's shown.
- **Reduced motion** (`MediaQuery.disableAnimations`): no confetti; the review
  timer still runs (it's not an animation). The widget tests run under reduced
  motion and pump the 1.2 s explicitly.
