# RevenueCat core — SDK, purchase service, cached premium state

**Prompt:** `claude-prompts/2026-06-12/002-revenuecat-core.md`
**Completed:** 2026-06-12 · **Status:** done

## Summary

Added the RevenueCat plumbing the later paywall prompts will sit on — no UI and
no feature gating yet. The app now has a `PurchaseService` that knows whether the
user is premium, seeded instantly from an on-device cache (so it works offline
and on the first frame) and kept live by RevenueCat's customer-info listener.
Two Riverpod providers expose it (`purchaseServiceProvider`, `isPremiumProvider`),
`main.dart` configures the SDK in the background without blocking startup, and the
whole thing is mockable in tests via a reusable fake. Analyze is clean, all 55
tests pass, and a debug Android APK builds with the SDK linked in.

## Decisions

- **Abstract `PurchaseService` + concrete RevenueCat impl + fake.** The interface
  lives in `lib/` so later paywall prompts and tests can inject a fake with
  `purchaseServiceProvider.overrideWithValue(...)`. The fake lives in `test/support/`
  so it's shared but doesn't bloat the shipped app.
- **Named the result enum `PurchaseOutcome`, not `PurchaseResult`.** RevenueCat
  10.x already exports a `PurchaseResult` type; reusing the name would collide.
  `success` / `cancelled` / `error` — cancellation is explicitly *not* an error.
- **Premium seeded synchronously in the constructor** from `prefs.premiumCached`
  (or `true` on unsupported platforms). So `isPremiumProvider` has the right answer
  on the very first frame, before any network call — the offline-first requirement.
- **Platform support is injectable** (`RevenueCatPurchaseService(prefs,
  supported: ...)`). Real builds auto-detect (Android/iOS/macOS supported; Windows/
  Linux/web/desktop-dev → premium unlocked per CLAUDE.md §3). The override lets the
  unit tests exercise both paths without real SDK calls.
- **Used the non-deprecated `Purchases.purchase(PurchaseParams.package(...))`** —
  `purchasePackage` is deprecated in 10.x.
- **Every SDK call is wrapped in try/catch.** A placeholder key or a network
  failure logs and continues with the cached status; it never crashes or blocks.
